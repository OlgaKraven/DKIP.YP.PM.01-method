## 1. Назначение и цели учебной практики

Учебная практика по ПМ.01 направлена на формирование у студентов базовых навыков разработки модулей программного обеспечения на языке C#, включая:

- работу с переменными, типами данных, операторами и условными конструкциями;
- использование циклов, массивов и коллекций для обработки данных;
- структурирование кода с помощью методов и простых моделей данных (ООП-минимум);
- реализацию простых алгоритмов (сортировка, поиск, фильтрация);
- базовую отладку и обработку ошибок ввода.

Цель практики — чтобы студент смог:
- понять фундаментальные конструкции языка C#;
- спроектировать и реализовать простой модуль обработки данных в виде консольного приложения;
- оформить результаты работы в виде отчёта с тестами и скриншотами.

## 2. Структура методических материалов

Методические материалы структурированы по разделам и связаны единой навигацией с главной страницы `index`:

### 2.1. Главная страница (index)

На главной странице представлены:

- ссылки на раздел **«1. Методические материалы по C#»** и **«2. Выполнение задания»**;

Рекомендуется начинать работу с чтения описания на `index`, затем переходить к теории (раздел 1) и практической части (раздел 2).

### 2.2. Раздел 1. Методические материалы по C#

Раздел посвящён основам языка C# и включает:

- **Переменные и типы данных**  
  `int`, `double`, `string`, `bool`, `char`, `DateTime`, примеры объявления и использования.

- **Арифметические и логические операции, операторы сравнения**  
  `+`, `-`, `*`, `/`, `%`, `&&`, `||`, `!`, операторы `<`, `>`, `==`, `!=`, `<=`, `>=`.

- **Условные конструкции**  
  `if / else if / else`, `switch` для ветвления логики.

- **Циклы**  
  `for`, `foreach`, `while`, `do…while` для повторяющихся действий.

- **Массивы и коллекции**  
  - одномерные массивы `int[]`;  
  - списки `List<T>`: добавление, удаление, перебор и поиск по условию.

- **Методы и структурирование кода**  
  - создание методов с параметрами и возвращаемыми значениями;  
  - разделение логики по методам (не всё в `Main`);  
  - передача данных между методами;  
  - небольшой пример консольной программы, разбитой на функции.

- **Основы ООП (минимум)**  
  понятия «класс», «объект», поля, свойства, конструктор, простые модели данных (товар, задача, студент).

- **Работа со строками**  
  длина строки, изменение регистра, поиск подстроки (`Contains`), разбиение строки (`Split`), преобразование строк в числа (`int.Parse`, `int.TryParse`).

- **Основы алгоритмов**  
  идея сортировки, поиска и фильтрации данных на простых примерах массивов и списков.

- **Простая отладка**  
  чтение сообщений компилятора, типовые ошибки, использование вывода в консоль для диагностики.

Этот раздел служит теоретической базой для выполнения практического задания.

### 2.3. Раздел 2. Выполнение задания

Практическая часть построена вокруг тестовой предметной области:

> **«Анализ температурного режима за неделю»**  
> Массив целых чисел описывает температуры по дням, над которым выполняются базовые операции обработки данных.

Ключевые элементы задания:

- описание предметной области и примеры массивов температур;
- постановка задач для студента (создание структуры программы с классами `ArrayProcessor` и `Program`);
- подготовка среды разработки (выбор IDE, проверка `dotnet --version`, создание консольного проекта `TemperatureAnalyzer`);
- проектирование структуры:
  - хранение данных в массиве целых чисел;
  - выбор методов для обработки (min, max, сумма, среднее, сортировки, фильтры и поиск).

Реализация включает:

- класс **`ArrayProcessor`** с методами:
  - ввод массива с клавиатуры с проверкой через `int.TryParse`;
  - генерация случайных данных в заданном диапазоне;
  - вывод массива;
  - вычисление минимума, максимума, суммы и среднего;
  - сортировка по возрастанию и убыванию;
  - линейный поиск;
  - формирование новых массивов: только положительные, только отрицательные, значения выше порога.

- класс **`Program`** с:
  - консольным меню (пункты 1–13 и 0 — выход);
  - основным циклом обработки выбора пользователя через `switch`;
  - использованием общей переменной `currentArray` для текущих данных;
  - вспомогательными методами `ReadInt` (безопасный ввод чисел) и `EnsureArrayCreated` (проверка наличия массива перед операциями).

- обработку некорректного ввода:
  - буквы вместо чисел в меню и при вводе температур;
  - выбор несуществующего пункта меню;
  - попытки работать с неинициализированным массивом.

- тестирование:
  - **Тест 1** — ручной ввод контролируемого массива `[-3, 0, 2, -1, 4, 1, -2]` с расчётом ожидаемых результатов;
  - **Тест 2** — случайная генерация массива с набором операций;
  - **Тест 3** — проверка устойчивости к ошибкам ввода (abc, пустые строки, неверный номер пункта, отсутствие массива).

- оформление отчёта:
  - описание предметной области;
  - описание структуры программы и реализованных методов;
  - таблица тестов (входные данные, операция, ожидаемый и фактический результат);
  - скриншоты работы программы;
  - выводы по результатам практики.

## 3. Рекомендуемый порядок работы студента

1. **Перейти на страницу методических материалов**  
   Открыть GitHub Pages:  
   [https://olgakraven.github.io/DKIP.YP.PM.01-method/](https://olgakraven.github.io/DKIP.YP.PM.01-method/)

2. **Изучить раздел 1 «Методические материалы по C#»**  
   Повторить базовые конструкции языка и примеры из теории, при необходимости выполнить небольшие фрагменты кода.

3. **Изучить раздел 2 «Выполнение задания»**  
   Внимательно прочитать описание предметной области, структуру программы и требования к методам `ArrayProcessor` и `Program`.

4. **Подготовить среду разработки**  
   Установить/проверить Visual Studio / Rider / VS Code + C#, проверить `dotnet --version`, создать консольный проект `TemperatureAnalyzer`.

5. **Реализовать программу**  
   - создать класс `ArrayProcessor` с необходимыми методами;
   - настроить консольное меню в `Program`;
   - обеспечить безопасный ввод и обработку ошибок.

6. **Протестировать программу**  
   Выполнить тесты, описанные в методичке, и зафиксировать результаты (желательно в таблице).

7. **Оформить отчёт**  
   Подготовить краткий отчёт: описание задачи, структура программы, список методов, тестовые примеры, скриншоты, выводы.

## 5. Ожидаемые результаты учебной практики

По завершении практики студент должен уметь:

- создавать консольные приложения на C# на основе .NET SDK;
- работать с массивами и коллекциями (`int[]`, `List<int>`) для решения прикладных задач;
- применять методы и простые ООП-конструкции для структурирования программы;
- реализовывать базовые алгоритмы обработки данных (сортировка, поиск, фильтрация);
- выполнять простую отладку кода и обрабатывать некорректный пользовательский ввод;
- оформлять результаты работы в виде отчёта с тестами и скриншотами.

Эти навыки составляют основу для дальнейшего изучения модулей по разработке и сопровождению программного обеспечения в рамках ПМ.01 и последующих профессиональных модулей.
