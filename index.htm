2. Основы языка C#

2.1. Переменные и типы данных

В C# каждая переменная имеет тип, который определяет, какой вид данных она хранит.

Тип Описание Пример
int целые числа int age = 20;
double дробные числа double price = 19.99;
string строки string name = "Alice";
bool логическое значение bool isActive = true;
char один символ char grade = 'A';
DateTime дата и время DateTime now = DateTime.Now;

Пример использования

int count = 5;
double temperature = 23.7;
string city = "Moscow";
bool isOpen = true;
char letter = 'Z';
DateTime today = DateTime.Today;

2.2. Арифметические операции

В C# поддерживаются стандартные арифметические операции:

Операция Пример Результат
+5 + 27
-9 - 36
*4 * 28
/10 / 33 (для int)
%10 % 31 (остаток)

Пример

int a = 10;
int b = 3;

int sum = a + b;       // 13
int diff = a - b;      // 7
int mult = a * b;      // 30
int div = a / b;       // 3
int mod = a % b;       // 1

2.3. Логические операции (&&, ||, !)

Используются для работы с булевыми значениями:

Оператор Описание Пример
&& логическое И true && false → false
|| логическое ИЛИ true || false → true
! логическое НЕ !true → false

Пример

bool isAdult = true;
bool hasTicket = false;

bool canEnter = isAdult && hasTicket;   // false
bool guestAccess = isAdult || hasTicket; // true
bool notAdult = !isAdult;                // false

2.4. Операторы сравнения

Используются в условиях:

Оператор Значение Пример
<меньше5 < 8 → true
>больше5 > 2 → true
==равно10 == 10 → true
!=не равно10 != 5 → true
<=меньше или равно3 <= 3
>=больше или равно5 >= 1

Пример

int x = 10;

bool isEqual = x == 10;   // true
bool isNotEqual = x != 5; // true
bool isLess = x < 5;      // false
bool isMore = x > 7;      // true

2.5. Условные конструкции

if / else if / else

int temp = 5;

if (temp > 20)
{
    Console.WriteLine("Тепло");
}
else if (temp > 0)
{
    Console.WriteLine("Прохладно");
}
else
{
    Console.WriteLine("Холодно");
}

switch

Используется при сравнении со значениями:

int day = 3;

switch (day)
{
    case 1:
        Console.WriteLine("Понедельник");
        break;
    case 2:
        Console.WriteLine("Вторник");
        break;
    case 3:
        Console.WriteLine("Среда");
        break;
    default:
        Console.WriteLine("Неизвестный день");
        break;
}

2.6. Циклы

for

Используется, когда известно количество повторений:

for (int i = 0; i < 5; i++)
{
    Console.WriteLine("i = " + i);
}

foreach

Перебор элементов коллекции:

string[] fruits = { "Apple", "Banana", "Cherry" };

foreach (string fruit in fruits)
{
    Console.WriteLine(fruit);
}

while

Цикл выполняется пока условие истинно:

int count = 3;

while (count > 0)
{
    Console.WriteLine(count);
    count--;
}

do…while

Выполняется хотя бы один раз:

int number = 1;

do
{
    Console.WriteLine("number = " + number);
    number++;
}
while (number <= 3);

3. Массивы и коллекции в C#

3.1. Одномерные массивы

Массив — это фиксированная по размеру структура данных, содержащая элементы одного типа.

Объявление массива

int[] numbers = new int[5];  // массив на 5 элементов

Инициализация массива

int[] nums = { 10, 20, 30, 40, 50 };

Доступ к элементам

Console.WriteLine(nums[0]); // 10
nums[2] = 99;               // элемент с индексом 2 станет 99

Перебор массива циклом for

for (int i = 0; i < nums.Length; i++)
{
    Console.WriteLine(nums[i]);
}

Перебор массива foreach

foreach (int item in nums)
{
    Console.WriteLine(item);
}

3.2. Списки (List<T>)

List<T> — динамическая коллекция, количество элементов может меняться.

Для работы нужен импорт:

using System.Collections.Generic;

3.2.1. Создание списка

List<int> scores = new List<int>();

Или с начальными значениями:

List<int> grades = new List<int> { 4, 5, 3, 5 };

3.2.2. Добавление элементов

scores.Add(10);
scores.Add(20);
scores.Add(30);

3.2.3. Удаление элементов

Удаление по значению:

grades.Remove(5); // удаляет первое найденное значение 5

Удаление по индексу:

grades.RemoveAt(0); // удаляет элемент с индексом 0

Очистка списка:

grades.Clear();

3.2.4. Перебор элементов списка

Через for:

for (int i = 0; i < scores.Count; i++)
{
    Console.WriteLine(scores[i]);
}

Через foreach:

foreach (int score in scores)
{
    Console.WriteLine(score);
}

3.2.5. Поиск по условию

Найти первый элемент, удовлетворяющий условию:

int found = grades.Find(g => g < 4); // ищем оценку < 4

Найти все элементы по условию:

List<int> goodGrades = grades.FindAll(g => g >= 4);

Проверка, есть ли элемент:

bool hasFive = grades.Contains(5);

Поиск индекса:

int index = grades.IndexOf(3);

4. Методы и структурирование кода в C#

Методы позволяют разбивать программу на небольшие, удобные части. Это делает код понятным, упрощает сопровождение и повторное использование.

4.1. Создание методов с параметрами

Метод — это блок кода, который выполняет определённое действие. Он может принимать параметры (входные данные).

Простой метод с параметром

void PrintMessage(string text)
{
    Console.WriteLine(text);
}

Вызов метода

PrintMessage("Привет, мир!");

Метод с несколькими параметрами

void ShowUserInfo(string name, int age)
{
    Console.WriteLine($"Имя: {name}, возраст: {age}");
}

ShowUserInfo("Аня", 20);

4.2. Возвращаемые значения (return)

Метод может возвращать значение, например число или строку.

Метод, который возвращает сумму

int Sum(int a, int b)
{
    return a + b;
}

int result = Sum(5, 3);
Console.WriteLine(result); // 8

Метод, возвращающий строку

string GetGreeting(string name)
{
    return $"Здравствуйте, {name}!";
}

Console.WriteLine(GetGreeting("Иван"));

4.3. Разделение логики по методам (не всё писать в Main)

Плохой подход

static void Main()
{
    int a = 5;
    int b = 7;
    int sum = a + b;
    Console.WriteLine(sum);
}

Лучший подход — вынести логику в отдельные методы

static void Main()
{
    int result = Sum(5, 7);
    PrintResult(result);
}

static int Sum(int a, int b)
{
    return a + b;
}

static void PrintResult(int value)
{
    Console.WriteLine("Результат: " + value);
}

Преимущества:

  • читабельность,
  • повторное использование,
  • легче тестировать,
  • проще сопровождать.

4.4. Передача данных между методами

Один метод может передавать данные другому методу через параметры и return.

Пример: два метода взаимодействуют

static void Main()
{
    int number = ReadNumber();
    int doubled = DoubleNumber(number);
    PrintResult(doubled);
}

static int ReadNumber()
{
    return 10;
}

static int DoubleNumber(int value)
{
    return value * 2;
}

static void PrintResult(int value)
{
    Console.WriteLine("Итог: " + value);
}

Поток данных: Main → ReadNumber → DoubleNumber → PrintResult

4.5. Пример мини-программы с методами

static void Main()
{
    int a = GetNumber("Введите первое число: ");
    int b = GetNumber("Введите второе число: ");

    int sum = Sum(a, b);

    PrintResult(sum);
}

static int GetNumber(string message)
{
    Console.Write(message);
    return int.Parse(Console.ReadLine());
}

static int Sum(int x, int y)
{
    return x + y;
}

static void PrintResult(int value)
{
    Console.WriteLine("Сумма = " + value);
}

5. Основы ООП (минимум)

ООП (объектно-ориентированное программирование) позволяет описывать сущности реального мира как структуры данных (классы) и создавать их экземпляры (объекты). Здесь рассматривается только минимум, нужный для алгоритмических и базовых учебных проектов.

5.1. Что такое класс

Класс — это шаблон (чертёж), по которому создаются объекты. Он описывает:

  • какие данные хранит объект (поля, свойства),
  • какие действия объект может выполнять (методы).

Пример класса

class Person
{
    public string Name;
    public int Age;
}

Это чертёж человека: указан набор данных, но реального “человека” пока нет.

5.2. Что такое объект

Объект — это конкретный экземпляр класса, созданный в памяти.

Создание объекта

Person user = new Person();
user.Name = "Анна";
user.Age = 20;

Console.WriteLine(user.Name);  // Анна

Класс — шаблон. Объект — конкретный пример.

5.3. Поля и свойства

Поля — хранят данные напрямую.

class Product
{
    public string Title;
    public double Price;
}

Свойства — более безопасная форма хранения данных (часто используются в реальных проектах).

class Product
{
    public string Title { get; set; }
    public double Price { get; set; }
}

Свойства можно использовать как обычные переменные:

Product p = new Product();
p.Title = "Телефон";
p.Price = 19999;

5.4. Конструктор класса

Конструктор — специальный метод, который вызывается при создании объекта. Он задаёт начальные значения.

Пример конструктора

class Student
{
    public string Name { get; set; }
    public int Group { get; set; }

    // Конструктор
    public Student(string name, int group)
    {
        Name = name;
        Group = group;
    }
}

Создание объекта

Student s = new Student("Иван", 35);
Console.WriteLine($"{s.Name}, группа {s.Group}");

5.5. Простые модели данных

Модель данных — это класс, который описывает сущность предметной области.

Пример: модель товара

class Product
{
    public string Title { get; set; }
    public double Price { get; set; }
}

Пример: модель задачи

class TaskItem
{
    public string Description { get; set; }
    public bool IsCompleted { get; set; }
}

Пример: модель студента

class Student
{
    public string FullName { get; set; }
    public int Course { get; set; }
}

Такие модели используются:

  • при работе с коллекциями,
  • при обмене данными,
  • для хранения результатов программ.

5.6. Пример мини-программы с ООП

class Program
{
    static void Main()
    {
        Product apple = new Product("Яблоко", 50);
        Product banana = new Product("Банан", 60);

        PrintProduct(apple);
        PrintProduct(banana);
    }

    static void PrintProduct(Product p)
    {
        Console.WriteLine($"{p.Title}: {p.Price} руб.");
    }
}

class Product
{
    public string Title { get; set; }
    public double Price { get; set; }

    public Product(string title, double price)
    {
        Title = title;
        Price = price;
    }
}

6. Работа со строками в C#

Строки (string) — один из самых часто используемых типов данных. В C# строки имеют множество полезных методов для обработки текста.

6.1. Основные методы строк

6.1.1. Длина строки — Length

Возвращает количество символов в строке.

string text = "Hello";
int len = text.Length; // 5

6.1.2. Преобразование регистра

ToLower() — весь текст в нижний регистр:

string title = "HELLO";
Console.WriteLine(title.ToLower()); // "hello"

ToUpper() — весь текст в верхний регистр:

string city = "Moscow";
Console.WriteLine(city.ToUpper()); // "MOSCOW"

6.1.3. Проверка содержания — Contains()

Возвращает true, если строка содержит определённый фрагмент.

string message = "Добро пожаловать";

bool hasWord = message.Contains("пожа"); // true

Можно сделать регистронезависимую проверку:

bool hasCity = "MOSCOW".ToLower().Contains("mos");

6.1.4. Разделение строки — Split()

Разделяет строку на части и возвращает массив.

string data = "apple,banana,orange";
string[] fruits = data.Split(',');

foreach (var f in fruits)
{
    Console.WriteLine(f);
}

// apple
// banana
// orange

Можно разделять по пробелам:

string text = "one two three";
string[] words = text.Split(' ');

6.2. Преобразование строк в числа

6.2.1. int.Parse()

Преобразует строку в int. Если строка содержит не число, программа выдаст ошибку.

string input = "123";
int number = int.Parse(input); // 123

6.2.2. int.TryParse() (желательно)

Позволяет безопасно преобразовать строку без исключения. Возвращает:

  • true, если преобразование удалось,
  • false, если нет.

Пример

string input = "45";
if (int.TryParse(input, out int value))
{
    Console.WriteLine("Число = " + value);
}
else
{
    Console.WriteLine("Ошибка: это не число");
}

Ошибка не вызовет аварийного завершения программы.

6.3. Пример мини-программы со строками

Console.Write("Введите числа через запятую: ");
string input = Console.ReadLine();                // "10,20,30"

string[] parts = input.Split(',');                // ["10","20","30"]

int sum = 0;

foreach (string part in parts)
{
    if (int.TryParse(part, out int num))
    {
        sum += num;
    }
}

Console.WriteLine("Сумма = " + sum);

7. Основы алгоритмов

Алгоритм — это последовательность действий, выполняемых для решения задачи. В рамках базового уровня студенту достаточно понимать идею работы нескольких ключевых алгоритмов: сортировка, поиск и фильтрация данных.

7.1. Что такое сортировка

Сортировка — это процесс упорядочивания данных по определённому правилу:

  • по возрастанию,
  • по убыванию,
  • по алфавиту.

Важно понимать идею, а не конкретные реализации.

Идея сортировки:

  • есть набор данных,
  • мы хотим расположить их в порядке от меньшего к большему (или наоборот),
  • алгоритм «переставляет» элементы.

Пример сортировки в C# (сортировка целых чисел)

int[] numbers = { 5, 2, 9, 1 };

Array.Sort(numbers); // сортировка по возрастанию

foreach (int n in numbers)
{
    Console.WriteLine(n);
}

Результат:

1
2
5
9

Список сортировок: Bubble sort, QuickSort, MergeSort — знать не обязательно, только понимать идею.

7.2. Что такое поиск

Поиск — это нахождение элемента в наборе данных.

Идея поиска:

  • есть набор значений,
  • мы ищем нужный элемент,
  • алгоритм проверяет значения одно за другим.

Пример линейного поиска (самый простой вариант)

int[] nums = { 10, 20, 30, 40 };
int target = 30;

bool found = false;

foreach (int n in nums)
{
    if (n == target)
    {
        found = true;
        break;
    }
}

Console.WriteLine(found); // true

Что нужно знать:

  • поиск проверяет элементы,
  • поиск может быть медленным или быстрым,
  • студент должен уметь найти элемент в массиве или списке.

7.3. Что такое фильтрация данных

Фильтрация — это выборка элементов, которые подходят под условие.

Идея фильтрации:

  • есть коллекция значений,
  • мы выбираем только те элементы, которые удовлетворяют правилу.

Например:

  • выбрать только положительные числа,
  • отобрать строки, содержащие нужное слово,
  • оставить товары с ценой больше 100.

Пример фильтрации списка

List<int> data = new List<int> { 1, -5, 7, 0, 3, -2 };

List<int> positive = data.FindAll(x => x > 0);

foreach (int n in positive)
{
    Console.WriteLine(n);
}

Результат:

1
7
3

Объединённый пример: сортировка + поиск + фильтрация

List<int> nums = new List<int> { 7, 2, 10, -3, 5 };

// 1. Фильтрация: оставить только положительные
List<int> positive = nums.FindAll(x => x > 0);

// 2. Сортировка
positive.Sort();

// 3. Поиск
bool hasFive = positive.Contains(5);

Console.WriteLine("Отфильтрованные: " + string.Join(", ", positive));
Console.WriteLine("Есть ли 5? " + hasFive);

8. Простая отладка (начальный уровень)

Отладка — это процесс поиска и исправления ошибок в программе. Студент должен уметь разбираться в сообщениях компилятора, запускать программу, проверять её работу и исправлять логические ошибки.

8.1. Понимание сообщений об ошибках компиляции

При запуске программы компилятор может сообщить об ошибке. Обычно в сообщении указано:

  • тип ошибки,
  • строка,
  • короткое описание,
  • предполагаемое место, где ошибка возникла.

Пример ошибки:

error CS1002: ; expected

Это означает, что пропущена ;.

Пример ошибки:

error CS0029: Cannot implicitly convert type 'string' to 'int'

Значит: пытаешься присвоить строку числу.

Пример ошибки:

error CS0103: The name 'valuee' does not exist in the current context

Обычно означает: опечатка в имени переменной.

Важно: компилятор указывает строку примерно — иногда проблема выше.

8.2. Умение запускать и проверять работу программы

Студент должен:

  • Запускать программу
    • в Visual Studio — кнопка Start или F5;
    • в консоли — dotnet run.
  • Вводить тестовые данные
    • проверять разные случаи: правильные значения, неправильные значения, крайние значения (0, мин/макс).
  • Читать вывод
    • сравнивать фактический результат с ожидаемым.

8.3. Исправление логических ошибок

Логическая ошибка — программа компилируется, но работает неправильно.

Примеры логических ошибок:

Ошибка в формуле:

int a = 10;
int b = 5;
int result = a - b; // хотели сложить — сложили неправильно

Неверное условие:

if (age > 18)  // неправильно для "18 лет можно"
{
    // ...
}

Ошибки в цикле:

for (int i = 0; i <= 5; i++)
{
    // повторит лишний раз
}

Неверный поиск:

if (name.Contains("ivan"))  // забыли сравнение без регистра
{
    // ...
}

8.4. Пример простого процесса отладки

Исходный код (с ошибками)

int a = 10;
int b = 0;

int result = a / b; // Ошибка: деление на ноль

Console.WriteLine(reslt); // Ошибка: опечатка

Шаги отладки

  1. Компилятор сообщает: 
    error CS0103: The name 'reslt' does not exist
    Исправляем reslt → result.
  2. Программа падает при запуске: 
    System.DivideByZeroException
    Исправляем логику: 
    if (b != 0)
    Console.WriteLine(a / b);
else
    Console.WriteLine("Ошибка: деление на ноль!");
  3. Тестируем еще раз.

8.5. Использование вывода для поиска ошибок

Студент должен уметь выводить промежуточные значения:

Console.WriteLine("Debug: value = " + value);

Это помогает найти место, где программа работает не так, как ожидалось.
