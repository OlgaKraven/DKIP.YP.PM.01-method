2.1. Описание тестовой предметной области

Тестовая предметная область: «Анализ температурного режима за неделю».

Программист работает с набором данных, представляющим суточные температуры воздуха за 7 дней. Каждый элемент массива — температура в градусах Цельсия (целое число), измеренная один раз в сутки.

Описание предметной области

Информационная система должна уметь анализировать недельные температурные показатели. На основе массива температур выполняются базовые операции обработки данных.

Примеры значений массива

[-3, 0, +2, -1, +4, +1, -2]

Функциональные возможности программы

  • Генерация массива случайных температур за неделю (от –20 до +20).
  • Ввод массива вручную (студент сам вводит 7 значений).
  • Поиск минимальной и максимальной температуры недели.
  • Расчёт:
    • суммы — суммарное отклонение,
    • среднего — средняя температура недели.
  • Сортировка температур (например, для нахождения тренда).
  • Поиск конкретного значения (например, было ли 0°C на неделе).
  • Формирование нового массива:
    • только положительные температуры,
    • только отрицательные температуры,
    • температуры выше заданного порога.

2.5.2. Метод создания массива вручную

Задача: пользователь вводит температуры с клавиатуры, ввод проверяется через int.TryParse, при ошибке — повторный запрос.

Код для добавления в класс ArrayProcessor:

public int[] CreateArrayFromUserInput()
{
    Console.Write("Введите количество элементов массива (например, 7): ");

    int length;
    while (!int.TryParse(Console.ReadLine(), out length) || length <= 0)
    {
        Console.WriteLine("Ошибка. Введите положительное целое число.");
        Console.Write("Введите количество элементов массива: ");
    }

    int[] array = new int[length];

    Console.WriteLine($"Введите {length} целых значений (например, температуры в °C):");

    for (int i = 0; i < length; i++)
    {
        Console.Write($"Элемент [{i}]: ");
        int value;

        while (!int.TryParse(Console.ReadLine(), out value))
        {
            Console.WriteLine("Ошибка. Введите целое число.");
            Console.Write($"Элемент [{i}]: ");
        }

        array[i] = value;
    }

    return array;
}

2.5.3. Метод генерации массива случайных значений

Задача: сгенерировать массив целых чисел в заданном диапазоне (например, от –20 до +20).

Код для добавления в класс ArrayProcessor:

public int[] CreateRandomArray(int length, int minValue, int maxValue)
{
    if (length <= 0)
    {
        throw new ArgumentException("Длина массива должна быть положительной.");
    }

    if (minValue > maxValue)
    {
        throw new ArgumentException("Минимальное значение не может быть больше максимального.");
    }

    int[] array = new int[length];
    Random random = new Random();

    for (int i = 0; i < length; i++)
    {
        array[i] = random.Next(minValue, maxValue + 1); // верхняя граница не включается
    }

    return array;
}

2.5.5. Методы min и max

Задача: найти минимальное и максимальное значение в массиве.

Код для добавления в класс ArrayProcessor:

public int FindMin(int[] array)
{
    if (array == null || array.Length == 0)
    {
        throw new ArgumentException("Массив пуст или не инициализирован.");
    }

    int min = array[0];

    for (int i = 1; i < array.Length; i++)
    {
        if (array[i] < min)
        {
            min = array[i];
        }
    }

    return min;
}

public int FindMax(int[] array)
{
    if (array == null || array.Length == 0)
    {
        throw new ArgumentException("Массив пуст или не инициализирован.");
    }

    int max = array[0];

    for (int i = 1; i < array.Length; i++)
    {
        if (array[i] > max)
        {
            max = array[i];
        }
    }

    return max;
}

2.5.6. Методы суммы и среднего значения

Задача: посчитать сумму всех элементов и среднее значение.

Код для добавления в класс ArrayProcessor:

public int CalculateSum(int[] array)
{
    if (array == null || array.Length == 0)
    {
        throw new ArgumentException("Массив пуст или не инициализирован.");
    }

    int sum = 0;

    for (int i = 0; i < array.Length; i++)
    {
        sum += array[i];
    }

    return sum;
}

public double CalculateAverage(int[] array)
{
    if (array == null || array.Length == 0)
    {
        throw new ArgumentException("Массив пуст или не инициализирован.");
    }

    int sum = CalculateSum(array);
    double average = (double)sum / array.Length;

    return average;
}

2.5.7. Методы сортировки

Задача: отсортировать массив по возрастанию и по убыванию.

public void SortAscending(int[] array)
{
    if (array == null || array.Length == 0)
    {

    }

    Array.Sort(array);

public void SortDescending(int[] array)
{
    if (array == null || array.Length == 0)

    }

2.5.8. Метод линейного поиска

Задача: найти, встречается ли указанное значение в массиве, и вернуть его индекс.

Код для добавления в класс ArrayProcessor:

public int LinearSearch(int[] array, int value)
{
    if (array == null || array.Length == 0)
    {
        throw new ArgumentException("Массив пуст или не инициализирован.");
    }

    for (int i = 0; i < array.Length; i++)
    {
        if (array[i] == value)
        {
            return i; // индекс первого вхождения
        }
    }

    return -1; // если элемент не найден
}

2.5.9. Методы формирования нового массива по условию

Задача: отфильтровать элементы по условию и вернуть новый массив.

2.5.9.1. Только положительные значения

public int[] GetPositiveValues(int[] array)
{
    if (array == null || array.Length == 0)
    {
        throw new ArgumentException("Массив пуст или не инициализирован.");
    }

    // Сначала считаем количество подходящих элементов
    int count = 0;
    for (int i = 0; i < array.Length; i++)
    {
        if (array[i] > 0)
        {
            count++;
        }
    }

    // Создаём массив нужной длины
    int[] result = new int[count];
    int index = 0;

    // Заполняем его
    for (int i = 0; i < array.Length; i++)
    {
        if (array[i] > 0)
        {
            result[index] = array[i];
            index++;
        }
    }

    return result;
}

2.7. Обработка некорректного ввода

Цель: программа не должна «падать» при ошибках пользователя (буквы вместо чисел, неправильный выбор меню, работа без созданного массива).

2.7.1. Безопасный ввод пунктов меню

Идея:

Сделать отдельный метод ReadInt, который:

  • запрашивает ввод строки,
  • пытается преобразовать её в int через int.TryParse,
  • если ввод некорректный — выводит сообщение «Ошибка: введите номер пункта меню числом.» (или другое понятное),
  • повторяет запрос, пока не получит корректное число.

Этот же метод можно использовать и для других числовых вводов (порог, длина массива и т.д.).

Код для класса Program:

// Безопасный ввод целого числа для меню и других действий
private static int ReadInt(string prompt)
{
    while (true)
    {
        Console.Write(prompt);
        string? input = Console.ReadLine();

        if (int.TryParse(input, out int value))
        {
            return value;
        }

        // Сообщение при некорректном вводе
        Console.WriteLine("Ошибка: введите номер пункта меню числом.");
    }
}

Для других мест можно менять текст сообщения (например, «Ошибка: введите целое число»), но принцип тот же: TryParse + цикл.

2.7.2. Безопасный ввод чисел массива

Идея:

При вводе значений массива (температур) каждый элемент тоже нужно проверять через int.TryParse. Если пользователь ошибся — программа:

  • не записывает в массив «мусор»,
  • не падает,
  • просто просит ввести значение ещё раз.

Это уже реализовано в методе CreateArrayFromUserInput класса ArrayProcessor, но можно прямо в методичке показать этот пример как «образец безопасного ввода».

Код для класса ArrayProcessor (метод ввода массива вручную):

public int[] CreateArrayFromUserInput()
{
    Console.Write("Введите количество элементов массива (например, 7): ");

    int length;
    while (!int.TryParse(Console.ReadLine(), out length) || length <= 0)
    {
        Console.WriteLine("Ошибка: введите положительное целое число.");
        Console.Write("Введите количество элементов массива (например, 7): ");
    }

    int[] array = new int[length];

    Console.WriteLine($"Введите {length} целых значений (температуры в °C):");

    for (int i = 0; i < length; i++)
    {
        while (true)
        {
            Console.Write($"Элемент [{i}]: ");
            string? input = Console.ReadLine();

            // Здесь используется TryParse — безопасный ввод элемента массива
            if (int.TryParse(input, out int value))
            {
                array[i] = value;
                break; // выходим из внутреннего цикла, переходим к следующему элементу
            }
            else
            {
                Console.WriteLine("Ошибка: введите целое число для температуры.");
                // Цикл продолжится, значение заново запрашивается
            }
        }
    }

    return array;
}

Важно: ни при каких неправильных вводах программа не падает — она просто просит ввести значение ещё раз.

2.7.3. Проверка наличия массива перед операциями

Идея:

Перед тем как:

  • выводить массив,
  • сортировать,
  • искать значения,
  • вычислять min/max, сумму, среднее,

нужно проверить, что:

  • currentArray != null,
  • currentArray.Length > 0.

Если массив не создан — вывести сообщение: «Массив ещё не создан. Сначала введите или сгенерируйте данные.» и не выполнять алгоритм.

Для этого удобно сделать отдельный метод в Program — EnsureArrayCreated.

Код для класса Program:

// Проверка, что массив уже создан и не пустой
private static bool EnsureArrayCreated()
{
    if (currentArray == null || currentArray.Length == 0)
    {
        Console.WriteLine("Массив ещё не создан. Сначала введите или сгенерируйте данные.");
        return false;
    }

    return true;
}

Примеры использования:

private static void HandleShowMinTemperature()
{
    if (!EnsureArrayCreated())
        return; // выходим из метода, чтобы не было ошибок

    int min = processor.FindMin(currentArray!);
    Console.WriteLine($"Минимальная температура: {min}");
}

private static void HandleSortAscending()
{
    if (!EnsureArrayCreated())
        return;

    processor.SortAscending(currentArray!);
    Console.WriteLine("Массив отсортирован по возрастанию.");
    processor.PrintArray(currentArray!);
}

private static void HandleSearchTemperature()
{
    if (!EnsureArrayCreated())
        return;

    int value = ReadInt("Введите температуру для поиска: ");
    int index = processor.LinearSearch(currentArray!, value);

    if (index >= 0)
        Console.WriteLine($"Температура {value} найдена по индексу {index}.");
    else
        Console.WriteLine($"Температура {value} не найдена в массиве.");
}

2.8. Тестирование на примерах

Цель: убедиться, что все операции модуля работают корректно и программа устойчива к ошибкам ввода.

2.8.1. Тест 1 — ручной ввод (контролируемые данные)

Задача: протестировать логику на заранее известном массиве, где легко проверить руками.

2.8.1.1. Входные данные

Студент вручную вводит массив температур:

-3, 0, 2, -1, 4, 1, -2

Это можно сделать через пункт меню:

  1. Выбрать пункт 1. Ввести температуры вручную.
  2. Ввести длину массива: 7.
  3. Ввести по очереди элементы: -3, 0, 2, -1, 4, 1, -2.

2.8.1.2. Ожидаемые результаты по операциям

Можно оформить в таблице (для отчёта):

Операция Ожидаемый результат
Вывод массива -3 0 2 -1 4 1 -2
Минимальная температура -3
Максимальная температура 4
Сумма элементов -3 + 0 + 2 + (-1) + 4 + 1 + (-2) = 1
Среднее значение 1 / 7 ≈ 0.142857… (можно округлить до 0.14)
Сортировка по возрастанию -3 -2 -1 0 1 2 4
Сортировка по убыванию 4 2 1 0 -1 -2 -3
Поиск значения 2 Найден, индекс 2 (если считать с нуля)
Поиск значения 10 Не найден, индекс -1, сообщение «не найдена»
Положительные температуры 2 4 1
Температуры выше порога 0 2 4 1

В отчёте студент может указать: «Фактические результаты совпали с ожидаемыми» и приложить скриншоты.

2.8.1.3. Пример сценария работы в консоли

=== Анализ температур за период ===
1. Ввести температуры вручную
...
0. Выход

Выберите пункт меню: 1
Введите количество элементов массива (например, 7): 7
Введите 7 целых значений (температуры в °C):
Элемент [0]: -3
Элемент [1]: 0
Элемент [2]: 2
Элемент [3]: -1
Элемент [4]: 4
Элемент [5]: 1
Элемент [6]: -2

Текущий массив:
Элементы массива: -3 0 2 -1 4 1 -2

Далее студент последовательно:

  • выбирает пункт 4 → проверяет минимум,
  • 5 → максимум,
  • 6 → сумму,
  • 7 → среднее,
  • 8 и 9 → сортировки,
  • 10 → ищет 2, затем 10,
  • 11 → выводит положительные,
  • 13 → вводит порог 0 и проверяет результат.

Все эти шаги он может и должен снабдить скриншотами в отчёте.

2.8.2. Тест 2 — случайная генерация (проверка общего поведения)

Задача: убедиться, что программа корректно работает не только на «идеальном» учебном примере, но и на произвольных данных.

2.8.2.1. Входные данные

Студент:

  1. Выбирает пункт 2. Сгенерировать температуры случайно.
  2. Вводит, например:
    • длина массива: 7;
    • минимальное значение: -20;
    • максимальное значение: 20.

Пример, что программа может выдать (каждый запуск будет разный):

Текущий массив:
Элементы массива: 5 -2 0 1 -7 3 9

2.8.2.2. Проверяемые операции

На сгенерированном массиве студент повторяет основные действия:

  • вывод массива (3);
  • минимум (4);
  • максимум (5);
  • сумма (6);
  • среднее (7);
  • сортировка по возрастанию (8);
  • сортировка по убыванию (9);
  • поиск значения, которое точно есть (например, первого элемента массива);
  • поиск значения, которого скорее всего нет (например, 100);
  • фильтры:
    • положительные (11);
    • отрицательные (12);
    • выше порога (13, порог, например, 0 или 5).

2.8.2.3. Фиксация результатов в отчёте

  • Скриншот с исходным сгенерированным массивом.
  • Скриншот после сортировки по возрастанию.
  • Скриншот после сортировки по убыванию.
  • Скриншоты фильтров (положительные, отрицательные, выше порога).
  • Краткий вывод: «На случайно сгенерированных данных все операции выполняются без ошибок, результаты выглядят корректно.»

2.8.3. Тест 3 — проверка обработки ошибок (устойчивость)

Задача: специально «сломать» ввод, чтобы убедиться, что программа:

  • не вылетает с исключениями,
  • даёт понятные сообщения и корректно продолжает работу.

2.8.3.1. Неверный ввод числа в меню

Студент:

  1. Запускает программу.
  2. Вместо номера пункта меню вводит, например: abc.

Ожидаемое поведение (благодаря ReadInt с TryParse):

Выберите пункт меню: abc
Ошибка: введите номер пункта меню числом.
Выберите пункт меню:

Программа не падает, а просит ввести число ещё раз.

2.8.3.2. Пустая строка

В тех же местах (меню, длина массива, значение порога и т.п.) студент нажимает просто Enter.

Ожидаемое поведение:

  • TryParse возвращает false,
  • программа выводит сообщение об ошибке,
  • повторяет запрос.

2.8.3.3. Выбор несуществующего пункта меню

Студент:

  1. В меню вводит, например, 99.

Ожидаемое поведение (ветка default в switch):

Выберите пункт меню: 99
Ошибка: такого пункта меню не существует.

Программа:

  • не падает;
  • цикл продолжается;
  • после повторного ввода меню выводится снова.

2.8.3.4. Попытка работать с неинициализированным массивом

Студент:

  1. Запускает программу.
  2. Не создавая массива (не выбирая 1 или 2), сразу выбирает, например:
    • 3 — показать массив,
    • 4 — минимум,
    • 8 — сортировка.

Ожидаемое поведение (благодаря EnsureArrayCreated()):

Массив ещё не создан. Сначала введите или сгенерируйте данные.

И так для всех операций, которые используют currentArray.

2.8.4. Итоги тестирования в отчёте

  1. Краткое описание каждого теста:
    • Тест 1 — ручной ввод фиксированного массива.
    • Тест 2 — случайная генерация.
    • Тест 3 — проверка ошибок.
  2. Таблица тестов (пример):
    № теста Тип теста Входные данные Операции Ожидаемый результат / критерий Фактический результат
    1 Ручной ввод -3, 0, 2, -1, 4, 1, -2 min, max, sum, avg, sort, search Все значения совпадают с расчётами вручную Совпало
    2 Случайная генерация длина 7, диапазон -20…20 вывод, сортировки, фильтры Отсутствие ошибок, корректное выполнение алгоритмов Ошибок нет, логика корректна
    3 Ошибка ввода abc вместо номера меню выбор пункта меню Сообщение об ошибке, повтор ввода без вылета программы Сообщение выведено, меню повторно
    4 Нет массива сразу выбрать пункт 4 (min) минимум Сообщение «Массив ещё не создан…» Сообщение выведено, без ошибки
  3. Скриншоты:
    • из Теста 1 (последовательность операций),
    • из Теста 2,
    • из Теста 3 (с ошибочным вводом и сообщениями).
  4. Выводы:
    • программа корректно реализует все алгоритмы;
    • программа устойчива к ошибкам ввода пользователя;
    • требования задания (модуль обработки массива + консольное меню + защита от ошибок) выполнены.

2.9. Оформление отчёта

Цель: показать, что студент понимает, что он сделал.

2.9.1. Описание предметной области

Кратко описать:

  • что такое «температурный режим за неделю»,
  • что представляют собой данные (целые числа в °C),
  • почему анализ таких данных важен.

2.9.2. Описание структуры программы

Студент должен описать:

  • класс ArrayProcessor и его назначение,
  • класс Program и работу меню,
  • как данные передаются между методами.

2.9.3. Описание реализованных методов

Для каждого метода указать:

  • краткое назначение (что делает),
  • входные данные (параметры),
  • выходные данные (результат),
  • кратко — алгоритм (словами, без кода).

2.9.4. Тестовые примеры

Сделать таблицу:

№ теста Входной массив Операция Ожидаемый результат Фактический результат
1 -3, 0, 2, -1, 4, 1, -2 Минимум -3 -3
2 -3, 0, 2, -1, 4, 1, -2 Фильтр > 0 2, 4, 1 2, 4, 1

2.9.5. Скриншоты

Студент должен:

  • сделать несколько скриншотов консольного окна:
    • меню,
    • пример сортировки,
    • пример поиска,
    • пример фильтрации,
    • пример обработки ошибки ввода,
  • вставить их в отчёт.

2.9.6. Выводы

Студент формулирует:

  • что он узнал,
  • какие алгоритмы реализовал,
  • как программа может быть расширена (например, хранение данных за месяц).
